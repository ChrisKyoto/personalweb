Solution Prioritization & Validation

Project Background

In this case we as a student of RevoU program, got study case scenario to measured solution prioritization for logistics company called GerakCepat. GerakCepat considered as an on going logistics company who want to solved their problems for internal user and increasing their revenue.

Problem

I got task to prioritized products features from certain conditions. Here are elaboration for conditions. First, we want to add same day delivery service in same cities from seller to buyer. Second, adding barcode in every package for automatically detecting package. Third, improve UX/UI Ops Dashboard for internal usage. Fourth, add annual subscriptions for package delivery. These are given conditions from RevoU.

Solution

In the beginning I already given situation and solutions to prioritized. Method that I'm using in this project are Kano Model, & Cost of Delay prioritization framework. I chose more than 1 framework because I feel in this project we need Kano Model for knowing users prespective on product development, and Cost of Delay to measured how the product will impact if we delayed the developing process.

Process

In this project I made Solutions Prioritization, Product Roadmap, and Product Backlog.
